Exhelp

EXtendable HELPer Class/Object/Framework for Javascript

View on GitHub Download .zip Download .tar.gz

Requirements

You're gonna need a browser (exHelp might work in Node.js, but I cannot confirm that it does). Having a text editor might be useful as well, or maybe you are just that fancy that you even use an IDE like VisualStudio, in which case I have great news: All functions have documentation comments, so autocompletion should work!

Setup

Add the "exHelp.js" or "exHelp.min.js" to your document via script tags

<script src="/js/lib/exHelp.min.js" type="text/javascript"></script>

you can also load it asynchronously (just make sure that jQuery is loaded synchronously, will be fixed to support complete async loading in next patch)

<script async src="/js/lib/exHelp.min.js" type="text/javascript"></script>

If you want to make use of the Date Helper plugin, also load the "exHelp_Date[.min].js" file, it shouldn't matter where you load it, since it hooks asynchronously to exHelp, but just to be sure do it after the main file. Same goes for the DOM Plugin which can be found under the "exHelp_Element[.min].js" file.

Now you should have the exHelp available and you can create a shortcut to exHelp like so:

var app = window.exHelp;

Now all exHelp functionality is accessible via the app object. For example:

var myStorage = app.storage("temp_data"); // Create a new storage object
// save data to storage
myStorage.username = "test";
myStorage.password = app.random.string(6); // Get a random string of length 6

Extending

You can also add your own functions to exHelp. Assume we have got an object with a few functions:

var o = {
    times: function(num, multi)
    {
        return num * multi;
    }
};

We can now add this function to exHelp:

app.extend(o);

and then you can simply call

app.times(2, 4); // = 8

Best practice

In my opinion (or at least, this is how I do it) put all your functions inside exHelp, this will make everything neat and organized, you can even add your own "subclasses" to keep everything categorized.

Some examples:

var o = {
    custom: {
        myFnc1: function(e)
        {
            return e * e;
        },
        myFnc2: function(e)
        {
            // You can use this.
            return this.myFnc1(e) * e;
        },
        myFnc3: function(e)
        {
            // Or you can use your exHelp shortcut, if you have set it up as a global variable
            return app.custom.myFnc2(e) * e;
        }
    }
};
app.extend(o);

now you can easily access you new functions via:

app.custom.myFnc1(2);
app.custom.myFnc2(4);
app.custom.myFnc3(8);

Of course, this is a rudimentary way of setting up your exHelp. The graceful (and mobile friendly) way would be to load your "setup" code in a self-executing function within a script file

(function MyAppSetup(window)
{
})(window);

If you don't know, this creates the function MyAppSetup within the scope of the parenthesis and then executes it since we have a trailing () after the parenthesis that hold the function. We pass the window variable for consistency. All of this ensures that the function will be executed as soon as the javascript file is done loading as well as keeping all local variables enclosed within the scope of our "quasi-constructor".

Best case you want the file to even load asynchronously (so it doesn't block rendering of the websites, e.g. on mobile devices), so we add this little tidbit at the top of our constructor after adding the async attribute on the script tag:

(function MyAppSetup(window)
{
    if (window['exHelp'] == void 0)
        return setTimeout(MyAppSetup, 100, window);
})(window);

This checks whether exHelp exists in the global window (We compare to void 0 instead of undefined and use the array annotation instead of object to avoid fatal errors during script execution which would block everything else from working.

So if exHelp doesn't exist globally (which it exposes itself to after initialization), it stops execution right there, then tries again in 100 ms (which in most cases is more than enough), which also explains why we don't use an anonymous function here. We need the name to be able to call it again.

So now we have to different paths we can go, either you want to make an exHelp Plugin (Like exHelp Date Helper) or you want to actually develop your system with it, depending on which you want to do, we have to do slightly different things (but easy enough to remember) - but remember, the plugin way always works!

Plugin

(function MyPluginSetup(window)
{
    if (window['exHelp'] == void 0)
        return setTimeout(MyPluginSetup, 100, window);

    var myPlugin =
        {
            myFunction: function (a, b)
            {
                return parseInt(a) * parseInt(b);
            },
            anotherFunction: function(a)
            {
                return myPlugin.myFunction(a, a * 2);
            }
        };

    window.exHelpExtend(myPlugin);
})(window);

You create your object that holds all functions of your plugin, then extend the exHelp base object.

Within a plugin, you either back-reference with the this keyword or if you need a function from a different scope of your plugin, you use the object name that you are initializing it with.

System

(function MyAppSetup(window)
{
    if (window['exHelp'] == void 0)
        return setTimeout(MyAppSetup, 100, window);

    var myapp = new exHelp();

    // Extending is optional, but recommended
    var baseExtension =
        {
            info:
                {
                    AppName: "My App",
                    Version: "1.0.0",
                    Author: "Leroy Jenkins"
                },

            myFunction: function (a, b)
            {
                return parseInt(a) * parseInt(b);
            }
        };

    myapp.extend(baseExtension);

    // Expose
    window.m = myapp;
})(window);

So you want to actually USE exHelp? Follow the design outlined above. You expose exHelp into a variable - optionally you can also extend your exHelp object with your own functions here (which you will want to do) and then extend them.
